# HR-Analytics-Dashboard-using-Excel
HR Analytics Dashboard using Excel
This project is a practical attempt to understand how HR data can be transformed into meaningful insights using Microsoft Excel. The idea behind building this dashboard was to simulate a real-world HR analytics scenario where decision-makers need quick, clear, and actionable information about their workforce.

Instead of working with overly complex tools, this project focuses on Excel as it is still one of the most widely used tools in HR functions across organizations. The dashboard is designed to be simple, interactive, and easy to interpret, even for users who may not have a technical background.

**Project Overview**

The dashboard brings together different HR metrics into a single view, allowing users to track key indicators such as employee count, attrition, department-wise distribution, and other workforce trends. It is built using Excel features like Pivot Tables, Charts, Slicers, and basic formulas.

The goal is not just to visualize data, but to tell a story — for example:

Are employees leaving more from a specific department?
Is there a pattern in attrition based on age or experience?
How is the workforce distributed across different roles?

By answering such questions, the dashboard helps in supporting better HR decision-making.

**Key Features**
Interactive filters (slicers) to explore data dynamically
Department-wise and role-wise employee distribution
Attrition analysis to identify trends and patterns
Clean and structured layout for easy understanding
Visual representation using charts instead of raw numbers
Assumptions

While building this dashboard, a few assumptions were made about the dataset:

The data provided is accurate and free from major inconsistencies
Each row represents one employee record
Attrition is recorded as a binary value (e.g., Yes/No)
Departments, job roles, and other categories are predefined and consistent
Time-based analysis (like monthly trends) is limited if date fields are not detailed

These assumptions are important because any variation in the dataset structure can affect how the dashboard behaves.

**Limitations**

This project also has some limitations, which are worth noting:

The dashboard is built in Excel, so it may not handle very large datasets efficiently
Real-time data updates are not supported (data needs to be refreshed manually)
Limited advanced analytics (e.g., predictive modeling or machine learning)
Insights depend heavily on the quality and completeness of the dataset
Some HR factors like employee satisfaction or performance may not be fully captured if not included in the data

**Learning Outcome**

Working on this project helped in understanding how raw HR data can be cleaned, structured, and visualized effectively. It also highlights the importance of asking the right questions while analyzing data, rather than just focusing on tools.

Overall, this dashboard serves as a foundational step towards more advanced HR analytics, and it can be further enhanced using tools like Power BI, SQL, or Python for deeper analysis.
